Development

System diagram

HoW system will be built?

  • Download the minimal ISO from Ubuntu.com to get basic kernel with some utilities.
  • Configure system locale and keyboard.
  • Download source tarballs of essential utilities like apt,python,libraries, etc.
  • Install tarballs.
  • Configure installations like paths etc.
  • It is still command line... NO GUI!

DESKTOP ENVIRONMENTS

  • Most famous D.E. in the market.
          1.Unity UI
          2.GNOME Shell
          3.XFCE UI
          4.LXDE UI
  • We have choosen GNOME Shell for DMLinux and XFCE UI for OpenGujarat based on our analysis.

    • HOW SYSTEM FILES WILL BE CHANGED?

    • We will use following tools to delete/add/modify system files.
      1.chroot utility
      2.UCK toolkit(ubuntu customization toolkit)
      3.Build essential toolkits
      4.Package management toolkits
      5.dconf or gconf editors to change system configurations
      6.init.d scripts to run some scripts at system start up



    DMLINUX

    (https://github.com/codejar-lab/dmlinux)

    Features of DMLINUX

    • Productive GUI(DE)
    • Language support like python,perl,ruby,etc.
    • Inbuilt essential IDEs like eclipse,netbeans,Qt
    • Markdown support
    • Customized DE with Eye-candy theme and icons
    • Inbuilt Web server (Apache,tomcat and Glassfish)
    • Aptana studio IDE for web developers
    • Customised Firfox with FirefoxOS for mobile phone
    • Linux app development utilities like quickly toolkit
    • ISO size : around 3 GB



    Opengujarat

    (https://github.com/codejar-lab/opengujarat)

    features of opengujarat

    • Developed in Gujarati
    • Very lightweight with XFCE DE
    • Essential Office apps with accounting software
    • Gujarati dictionary(developed by us) support
    • Simple GUI having Windows type mock-up
    • ISO size : around 900mb




    APP DEVELOPMENT

    APPlications

    • Auto ON Utility : Boot PC at given user time when PC is   already in shutdown.
    • Gujarati dictionary : English to Gujarati
    • LAMP front end : Apache MySQL and PHP front end to manage this running services
    • Repo-cloner : Local server package management utility
    • Multi Document converter : source markdown to multiple document format conversion
